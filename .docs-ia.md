## File naming convention (i18n)

Fumadocs `parser: 'dot'` (default): `{name}.mdx` is the default language (EN), `{name}.fr.mdx` is French. Same `meta.json` for both languages.

## meta.json pattern

```json
{
  "title": "Section title",
  "description": "Short hook for the section",
  "pages": ["index", "page1", "page2"]
}
```

## Page template (per page)

Every page follows the same beat structure:

```mdx
---
title: <Concrete value-led title>
description: <One sentence answering: what does this give me?>
---

<Lede paragraph — 2–3 sentences. State the problem, then state what this module solves.>

## TL;DR
- 3-5 bullets, scannable

## Install
<Gradle coordinate or "comes with nucleus-application">

## Quickstart
<Tiniest possible working snippet>

## How it works
<2-3 paragraphs of mental model — borrow the landing narrative tone>

## Reference
<API surface, organized by use case not by class>

## Troubleshooting / Notes
<Optional — only if real gotchas>
```

## Voice & tone

- **Marketing-tinted**: lift phrasing from the landing (e.g. "Cold start of a binary. Throughput of the JVM."). Each section opening should answer "why care?".
- **Senior dev energy**: no hand-holding on Gradle/Kotlin basics. Assume the reader ships software.
- **Show first**: every concept gets a snippet within the first scroll.
- **Avoid passive voice & hedging**. "Nucleus exposes" not "It may be possible to..."
- **No emojis** in body. Optional admonitions via `<Callout>`.
