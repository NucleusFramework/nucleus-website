Le plugin Gradle Nucleus (`id("dev.nucleusframework")`) expose une seule extension top-level, `nucleus`, pour configurer l'empaquetage, la signature, la publication et les builds natifs GraalVM depuis Kotlin. Elle s'appuie sur le DSL Compose Desktop de JetBrains et ajoute les formats, surfaces de signature et paramètres GraalVM propres à Nucleus. Cette page liste chaque propriété dans l'ordre où elle apparaît dans les sources du plugin.

## Top-level : `nucleus { }`

`dev.nucleusframework.NucleusExtension` :

- `val application: JvmApplication` — configuration de l'app JVM (`nucleus { application { … } }`).
- `val nativeApplication: NativeApplication` — cibles macOS Kotlin/Native.
- `val dependencies: NucleusPlugin.Dependencies` — coordonnées et notations des modules runtime.

## `application { }` — `JvmApplication`

- `from(SourceSet)` / `from(KotlinTarget)`
- `disableDefaultConfiguration()`
- `dependsOn(vararg tasks: Task | String)`
- `fromFiles(vararg files: Any)`
- `var mainClass: String?`
- `val mainJar: RegularFileProperty`
- `var javaHome: String`

### Enum `TargetFormat`

18 formats, dans l'ordre des sources :

`RawAppImage`, `Pkg`, `Deb`, `Rpm`, `Dmg`, `Exe`, `Msi`, `Nsis`, `NsisWeb`, `Portable`, `AppX`, `AppImage`, `Pacman`, `Snap`, `Flatpak`, `Zip`, `Tar`, `SevenZ`.

`RawAppImage` est l'app-image jpackage intermédiaire ; tous les autres formats sont produits par electron-builder. `Pkg`, `AppX` et `Flatpak` sont des formats store et exigent le sandboxing.

## `macOS { }` — `JvmMacOSPlatformSettings`

Étend `AbstractMacOSPlatformSettings`. Identification :

- `packageName: String?`, `packageVersion`, `packageBuildVersion`
- `dmgPackageVersion`, `dmgPackageBuildVersion`, `pkgPackageVersion`, `pkgPackageBuildVersion`
- `dockName: String?`, `setDockNameSameAsPackageName: Boolean = true`
- `bundleID: String?`, `appCategory: String?`, `minimumSystemVersion: String?`
- `installationPath: String? = null` — emplacement d'installation macOS ; quand il est absent, le build retombe sur `/Applications`
- `iconFile`, `layeredIconDir` (répertoire `.icon` macOS 26+)

Signature et sandbox :

- `entitlementsFile: RegularFileProperty`, `runtimeEntitlementsFile`
- `provisioningProfile: RegularFileProperty`, `runtimeProvisioningProfile`
- `signing { … }` — voir [Signature](#macos--signing--macossigningsettings)
- `notarization { … }` — voir [Notarisation](#macos--notarization--macosnotarizationsettings)

macOS 26 :

- `macOsSdkVersion: String? = "26.0"` — `vtool` patche le `LC_BUILD_VERSION` du launcher Mach-O pour activer Liquid Glass au SDK 26.

Composition :

- `launchAgents { agent(plistName) { bundleProgram; arguments; startInterval } }`
- `infoPlist { extraKeysRawXml }`
- `dmg { … }` — voir [DMG](#macos--dmg--dmgsettings)

### `linux.signing { }` — `LinuxSigningSettings`
Signe en GPG les DEB/RPM pour la distribution hors store. Voir [Signature de code](/docs/packaging/code-signing#linux--signature-gpg-des-debrpm).
- `enabled: Property<Boolean>` (défaut `false`)
- `keyId: Property<String>` — id de clé GPG, empreinte ou email
- `keyFile: RegularFileProperty` — clé privée ASCII-armored, importée dans un trousseau jetable (optionnel si la clé est déjà dans le trousseau de signature)
- `passphrase: Property<String>`
- `debMethod: DebSignMethod` (défaut `Detached` ; aussi `DpkgSig`, `Debsig`)
- `silentUpdate: Property<Boolean>` (défaut `false`) — auto-update sans mot de passe avec vérification de signature ; nécessite `enabled`. Voir [mises à jour sans mot de passe](/docs/packaging/auto-update#mises-à-jour-sans-mot-de-passe-sur-linux).

Les valeurs par défaut sont lues depuis `compose.desktop.linux.sign` et `compose.desktop.linux.signing.{keyId,keyFile,passphrase,silentUpdate}`.

- `publishMode: PublishMode` (`Never`, `Auto`, `Always`)

## GraalVM — `GraalvmSettings`

- `isEnabled: Property<Boolean>` (défaut `false`)
- `javaLanguageVersion: Property<Int>` (défaut `25`)
- `jvmVendor: Property<JvmVendorSpec>`
- `imageName: Property<String>`
- `march: Property<String>` (défaut `"native"`)
- `buildArgs: ListProperty<String>`
- `nativeImageConfigBaseDir: DirectoryProperty`
- `val macOS: GraalvmMacOSSettings` — `cStubsSrc`, `minimumSystemVersion = "12.0"`, `macOsSdkVersion = "26.0"`
- `val windows: GraalvmWindowsSettings` :
  - `bundleCRuntime: Property<Boolean> = true` — copie les DLL du runtime MSVC à côté du `.exe` natif
  - `dlls: ListProperty<String>` — défauts : `vcruntime140.dll`, `vcruntime140_1.dll`, `msvcp140.dll`
  - `sourceDir: DirectoryProperty` — répertoire d'où les DLL sont copiées (par défaut le `bin` de la toolchain GraalVM)
- `val metadataRepository: MetadataRepositorySettings` :
  - `enabled: Boolean = true`
  - `version: String = "0.10.6"`
  - `excludedModules: SetProperty<String>`
  - `moduleToConfigVersion: MapProperty<String, String>`

## Notes

Le plugin injecte une ressource `nucleus/nucleus-app.properties` dans l'app empaquetée, contenant `app.id`, `app.version`, `app.vendor`, `app.aumid`, `startup.task.id` et d'autres clés du même genre. L'objet `NucleusApp` les lit au démarrage : c'est ainsi que la configuration du DSL parvient à la bibliothèque runtime.
