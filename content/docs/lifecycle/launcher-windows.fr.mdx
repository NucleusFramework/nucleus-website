Le module `launcher-windows` expose quatre parties du shell Windows depuis Kotlin : les icônes
de superposition sur le bouton de la barre des tâches, les jump lists dans son menu contextuel,
les boutons de la miniature de la barre des tâches, ainsi que les badges numériques ou glyphes.
Chaque partie est un `object` distinct que tu appelles directement ; n'utilise que celles dont
tu as besoin.

## Ajouter la dépendance

```kotlin title="build.gradle.kts"
dependencies {
    implementation("dev.nucleusframework:nucleus.launcher-windows:<version>")
}
```

## Poser une icône de superposition

Une icône de superposition est un repère d'état de 16x16 dans le coin inférieur droit du bouton
de la barre des tâches. Passe la `Window` AWT et une source d'icône :

```kotlin
import dev.nucleusframework.launcher.windows.StockIcon
import dev.nucleusframework.launcher.windows.TaskbarIconSource
import dev.nucleusframework.launcher.windows.WindowsOverlayIcon

WindowsOverlayIcon.setIcon(
    window,
    TaskbarIconSource.FromStock(StockIcon.WARNING),
    description = "1 message non lu",
)

// plus tard
WindowsOverlayIcon.clearIcon(window)
```

Les icônes de superposition fonctionnent avec tous les formats de packaging. Chaque appel
renvoie `true` en cas de succès ; vérifie d'abord `WindowsOverlayIcon.isAvailable` si tu
supportes des plateformes autres que Windows.

## Construire une jump list

Une jump list regroupe des éléments dans le menu contextuel de la barre des tâches.
`setJumpList` remplace la liste entière de façon atomique :

```kotlin
import dev.nucleusframework.launcher.windows.JumpListCategory
import dev.nucleusframework.launcher.windows.JumpListItem
import dev.nucleusframework.launcher.windows.WindowsJumpListManager

WindowsJumpListManager.setJumpList(
    categories = listOf(
        JumpListCategory("Récents", items = listOf(
            JumpListItem("Projet A", arguments = "myapp://open?project=a"),
            JumpListItem("Projet B", arguments = "myapp://open?project=b"),
        )),
    ),
    tasks = listOf(
        JumpListItem("Nouvelle fenêtre", arguments = "myapp://new"),
        JumpListItem.SEPARATOR,
        JumpListItem("Préférences", arguments = "myapp://settings"),
    ),
)
```

Pour les apps non packagées (NSIS, MSI, distributable), appelle
`WindowsJumpListManager.setProcessAppId()` une seule fois tout au début de `main()`, avant
qu'aucune fenêtre ne soit créée. Les apps APPX/MSIX s'en passent : Windows utilise l'identité
du package.

<Callout type="info">
Quand l'utilisateur clique sur un élément de jump list, Windows lance un nouveau processus avec
les `arguments` de l'élément en ligne de commande. Combine-la avec
[`SingleInstanceManager`](/docs/lifecycle/single-instance) pour transférer ces arguments vers
l'instance déjà lancée.
</Callout>

## Ajouter des boutons de miniature

La barre d'outils de la miniature ajoute jusqu'à 7 boutons cliquables à l'aperçu de la miniature
de la barre des tâches. Enregistre-les une fois par fenêtre avec `setButtons` :

```kotlin
import dev.nucleusframework.launcher.windows.TaskbarIconSource
import dev.nucleusframework.launcher.windows.ThumbnailToolbarButton
import dev.nucleusframework.launcher.windows.WindowsThumbnailToolbar

WindowsThumbnailToolbar.setButtons(
    window,
    buttons = listOf(
        ThumbnailToolbarButton(id = 0, tooltip = "Précédent", icon = TaskbarIconSource.FromFile("icons/prev.ico")),
        ThumbnailToolbarButton(id = 1, tooltip = "Lecture",   icon = TaskbarIconSource.FromFile("icons/play.ico")),
        ThumbnailToolbarButton(id = 2, tooltip = "Suivant",   icon = TaskbarIconSource.FromFile("icons/next.ico")),
    ),
    onClick = { buttonId -> handleTransport(buttonId) },
)
```

Windows n'autorise pas l'ajout de boutons après le premier enregistrement. Pour modifier plus
tard l'icône, l'infobulle ou l'état d'un bouton, appelle `updateButtons(window, buttons)` avec
les mêmes id. Appelle `unregister(window)` à la fermeture de la fenêtre.

## Afficher un badge

Les badges affichent un compte numérique ou un glyphe d'état sur le bouton de la barre des
tâches et la tuile du menu Démarrer. Ils reposent sur un Application User Model ID que Windows
n'accorde qu'aux apps packagées, donc l'API de badge ne fonctionne que sous APPX/MSIX. Appelle
`initialize()` une fois avant de poser un badge :

```kotlin
import dev.nucleusframework.launcher.windows.BadgeGlyph
import dev.nucleusframework.launcher.windows.WindowsBadgeManager

WindowsBadgeManager.initialize()

WindowsBadgeManager.setGlyph(BadgeGlyph.NEW_MESSAGE)
// ou un compte numérique
WindowsBadgeManager.setCount(5)

// effacer
WindowsBadgeManager.clear()
```

`setCount` affiche 1 à 99 sous forme de nombre et 99+ au-delà ; `setCount(0)` l'efface. Appelle
`uninitialize()` à l'arrêt.

## Fonctionnement

Les icônes de superposition et la barre d'outils de miniature utilisent l'interface COM
`ITaskbarList3`, indexée par le HWND de ta `java.awt.Window`. COM est initialisé
paresseusement au premier appel, et les clics sur les boutons de miniature arrivent sur l'AWT
event dispatch thread.

Les jump lists utilisent `ICustomDestinationList`. Windows possède le menu, donc les clics
n'atteignent jamais ton processus en cours : l'OS démarre un nouvel exécutable avec les
`arguments` de l'élément ajoutés à la ligne de commande. C'est pourquoi les actions de jump
list transportent leurs données via `arguments` plutôt que par un callback.

Les badges utilisent WinRT et exigent un AUMID. Windows n'en accorde qu'aux apps packagées, donc
`WindowsBadgeManager` se déclare indisponible en dehors d'APPX/MSIX. L'AUMID est dérivé de
`NucleusApp.aumid`, sauf si tu passes une valeur explicite à `initialize`.

## Référence de l'API

### Icône de superposition

| Membre | Description |
|---|---|
| `WindowsOverlayIcon.setIcon(window, icon, description = "")` | Pose l'icône de superposition. Renvoie `Boolean`. |
| `WindowsOverlayIcon.clearIcon(window)` | Retire l'icône de superposition. Renvoie `Boolean`. |
| `WindowsOverlayIcon.isAvailable` | Indique si la bibliothèque native est chargée sur cette plateforme. |
| `WindowsOverlayIcon.lastError` | Message du dernier appel en échec, ou `null`. |

### Jump list

| Membre | Description |
|---|---|
| `WindowsJumpListManager.setProcessAppId(aumid = null)` | Définit l'AUMID du processus ; à appeler au début de `main()` pour les apps non packagées. |
| `WindowsJumpListManager.setJumpList(categories, tasks, knownCategories)` | Remplace la jump list entière. Tous les paramètres valent vide par défaut. |
| `WindowsJumpListManager.clearJumpList()` | Supprime toutes les entrées. |

Types associés :

| Type | Usage |
|---|---|
| `JumpListCategory(name, items)` | Un groupe nommé d'éléments. |
| `JumpListItem(title, arguments, description, icon, isSeparator)` | Une entrée cliquable ; tous les paramètres ont une valeur par défaut. |
| `JumpListItem.SEPARATOR` | Un séparateur visuel (tasks uniquement). |
| `KnownCategory.FREQUENT` / `KnownCategory.RECENT` | Listes gérées par le shell, peuplées depuis tes associations de fichiers. |

### Barre d'outils de miniature

| Membre | Description |
|---|---|
| `WindowsThumbnailToolbar.setButtons(window, buttons, onClick = null)` | Enregistre jusqu'à 7 boutons. Appelable une fois par fenêtre. |
| `WindowsThumbnailToolbar.updateButtons(window, buttons)` | Met à jour l'icône, l'infobulle et les flags des boutons déjà enregistrés. |
| `WindowsThumbnailToolbar.unregister(window)` | Retire le callback de clic et restaure la procédure de fenêtre. |

`ThumbnailToolbarButton(id, tooltip, icon, enabled, hidden, noBackground, dismissOnClick, nonInteractive)` —
`id` doit être compris entre 0 et 6, `tooltip` fait au plus 259 caractères. Le handler `onClick`
est un `ThumbBarClickListener` qui reçoit l'`id` du bouton cliqué.

### Badge

| Membre | Description |
|---|---|
| `WindowsBadgeManager.initialize(aumid = null)` | Prépare le sous-système de badge. Requis avant tout appel de badge ; APPX/MSIX uniquement. |
| `WindowsBadgeManager.setCount(count)` | Pose un badge numérique ; `0` l'efface. |
| `WindowsBadgeManager.setGlyph(glyph)` | Pose un glyphe d'état ; `BadgeGlyph.NONE` l'efface. |
| `WindowsBadgeManager.clear()` | Efface le badge. |
| `WindowsBadgeManager.uninitialize()` | Libère les ressources natives. |

Valeurs de `BadgeGlyph` : `NONE`, `ACTIVITY`, `ALARM`, `ALERT`, `ATTENTION`, `AVAILABLE`,
`AWAY`, `BUSY`, `ERROR`, `NEW_MESSAGE`, `PAUSED`, `PLAYING`, `UNAVAILABLE`.

### Sources d'icône

Toute API qui prend une icône accepte un `TaskbarIconSource` :

- `TaskbarIconSource.FromStock(stockIcon)` — l'une des 93 valeurs de `StockIcon` (`WARNING`,
  `ERROR`, `INFO`, `SHIELD`, `FOLDER`, ...). Aucun fichier requis.
- `TaskbarIconSource.FromFile(path)` — un fichier `.ico` sur disque.
- `TaskbarIconSource.FromResource(dllPath, index)` — un index d'icône dans une DLL de
  ressources comme `shell32.dll` ou `imageres.dll`.

## Notes

- Les éléments de jump list ont besoin d'un vrai exécutable. Ils ne se déclenchent pas sous
  `./gradlew run`, où le processus est `java.exe` — teste depuis `runDistributable` ou un build
  packagé.
- Pour une barre de progression dans le bouton de la barre des tâches lui-même, plutôt qu'une
  icône de superposition, utilise [`taskbar-progress`](/docs/lifecycle/taskbar-progress).
