description: linux-hidpi reads the Linux display scale factor through a JNI bridge and applies it before AWT initializes.
---

import { Callout } from 'fumadocs-ui/components/callout';

`linux-hidpi` detects the native display scale factor on Linux and applies it to the AWT
rendering pipeline. Stock OpenJDK ignores the desktop scale factor, so a Compose Desktop
window renders at 1x on a HiDPI screen. The module mirrors the detection logic of JetBrains
Runtime (`systemScale.c`) in a JNI bridge, so any JVM — including a GraalVM native image —
resolves the correct scale.

<Callout type="info">
JetBrains Runtime handles per-monitor scaling on its own. `linux-hidpi` is for stock OpenJDK
and GraalVM native image, where JBR is not available.
</Callout>

## Add the dependency

```kotlin title="build.gradle.kts"
dependencies {
    implementation("dev.nucleusframework:nucleus.linux-hidpi:<version>")
}
```

You can omit this dependency when you already depend on
[`graalvm-runtime`](/docs/performance/graalvm/runtime-bootstrap): it brings `linux-hidpi` in
transitively, and `GraalVmInitializer` calls `applyLinuxHiDpiScale()` for you.

## Apply the scale factor

Call `applyLinuxHiDpiScale()` before AWT initializes — that is, before `application {}`:

```kotlin
import androidx.compose.ui.window.application
import dev.nucleusframework.hidpi.applyLinuxHiDpiScale

fun main() {
    applyLinuxHiDpiScale()
    application {
        // your Compose Desktop app
    }
}
```

The function is a no-op on non-Linux platforms, so you can leave the call in place on macOS
and Windows.

To read the value yourself — for example to log it or feed it into another subsystem — call
`getLinuxNativeScaleFactor()` and set the property manually:

```kotlin
import androidx.compose.ui.window.application
import dev.nucleusframework.hidpi.getLinuxNativeScaleFactor

fun main() {
    if (System.getProperty("sun.java2d.uiScale") == null) {
        val scale = getLinuxNativeScaleFactor()
        if (scale > 0.0) {
            System.setProperty("sun.java2d.uiScale", scale.toString())
        }
    }

    application {
        // your Compose Desktop app
    }
}
```

## How it works

`getLinuxNativeScaleFactor()` consults six detection sources in priority order and returns
the first positive value it finds:

| Priority | Source | Description |
|----------|--------|-------------|
| 1 | `J2D_UISCALE` | Explicit JVM override (env var) |
| 2 | GSettings | GNOME `org.gnome.desktop.interface` → `scaling-factor` |
| 3 | Mutter DBus | GNOME fractional scale via `org.gnome.Mutter.DisplayConfig` |
| 4 | `GDK_SCALE` | GTK / GNOME session variable |
| 5 | `GDK_DPI_SCALE` | GTK fractional DPI multiplier |
| 6 | `Xft.dpi` | X Resource Manager (KDE, legacy GNOME, …) |

Detection runs in native code loaded through the `HiDpiLinuxBridge` JNI object. If the native
library fails to load — for example on a musl-only container without glibc — the function
falls back to parsing the `J2D_UISCALE`, `GDK_SCALE`, and `GDK_DPI_SCALE` environment
variables in pure Kotlin. When no source yields a value, it returns `0.0` and lets the JVM
decide.

`applyLinuxHiDpiScale()` runs when the detected scale is positive and `sun.java2d.uiScale` is
not already set. It first sets `GDK_SCALE` in the process environment through the native
bridge, which routes both rendering and mouse-coordinate scaling through the JDK's own
`X11GraphicsDevice` detection path — the same path JBR uses. As a fallback it also sets
`sun.java2d.uiScale.enabled` and `sun.java2d.uiScale`, so rendering is still scaled on JDKs
that do not read `GDK_SCALE`.

<Callout type="info">
AWT rounds to the nearest integer scale, so GNOME fractional scaling (125%, 150%) is
approximate. For per-monitor and fractional scaling on Linux, use the
[Tao backend](/docs/tao/hidpi), which talks to the compositor directly.
</Callout>

## API reference

| API | Description |
|-----|-------------|
| `fun getLinuxNativeScaleFactor(): Double` | Resolved scale factor (for example `1.0`, `1.25`, `2.0`). Returns `0.0` when detection fails or the platform is not Linux. |
| `fun applyLinuxHiDpiScale()` | Sets `sun.java2d.uiScale` (and `GDK_SCALE` via the native bridge) when the property is unset and the detected scale is greater than `0`. No-op on non-Linux platforms. |

Both functions live in the `dev.nucleusframework.hidpi` package.

## Notes

### Native binaries

The module ships pre-built shared libraries inside the JAR for Linux x64 and aarch64, both
named `libnucleus_linux_hidpi_jni.so`. `NativeLibraryLoader` from `core-runtime` extracts the
matching binary on first use.

### ProGuard

The Nucleus Gradle plugin adds this rule automatically:

```proguard
-keep class dev.nucleusframework.hidpi.HiDpiLinuxBridge {
    native <methods>;
}
```
