Les contrôles média te permettent de publier les métadonnées d'une piste dans le centre média
du système d'exploitation et de recevoir les commandes de lecture depuis Kotlin. Un seul objet
`MediaControlService` enveloppe l'interface D-Bus MPRIS sous Linux,
`MPNowPlayingInfoCenter` / `MPRemoteCommandCenter` sur macOS et les System Media Transport
Controls (SMTC) sur Windows.

## Ajouter la dépendance

```kotlin title="build.gradle.kts"
dependencies {
    implementation("dev.nucleusframework:nucleus.media-control:<version>")
}
```

## Publier les métadonnées de lecture

Vérifie que le backend natif est chargé, configure l'identité du lecteur, puis pousse les
métadonnées et l'état de lecture :

```kotlin
import dev.nucleusframework.media.control.MediaControlService
import dev.nucleusframework.media.control.MediaMetadata
import dev.nucleusframework.media.control.MediaPlaybackState
import dev.nucleusframework.media.control.MediaPlaybackStatus

if (MediaControlService.isAvailable()) {
    MediaControlService.configure()

    MediaControlService.setMetadata(
        MediaMetadata(
            title = "Strobe",
            artist = "deadmau5",
            album = "For Lack of a Better Name",
            coverUrl = "file:///tmp/cover.png",
            duration = 10 * 60 * 1_000L,
        ),
    )

    MediaControlService.setPlaybackState(
        MediaPlaybackState(status = MediaPlaybackStatus.PLAYING, positionMs = 0),
    )
}
```

`duration` et `positionMs` sont en millisecondes. Sous Linux, un `coverUrl` qui pointe vers un
fichier local doit utiliser le schéma `file://`.

## Gérer les commandes de lecture

Enregistre un listener avec `attach` pour recevoir les commandes du centre média de l'OS. Le
callback est dispatché sur l'EDT Swing, donc tu peux modifier l'état Compose ou Swing
directement :

```kotlin
import dev.nucleusframework.media.control.MediaControlEvent

MediaControlService.attach { event ->
    when (event) {
        MediaControlEvent.Play -> player.play()
        MediaControlEvent.Pause -> player.pause()
        MediaControlEvent.Toggle -> player.toggle()
        MediaControlEvent.Next -> player.skipToNext()
        MediaControlEvent.Previous -> player.skipToPrevious()
        MediaControlEvent.Stop -> player.stop()
        is MediaControlEvent.SeekBy -> player.seekBy(event.offsetMs)
        is MediaControlEvent.SetPosition -> player.seekTo(event.positionMs)
        else -> {}
    }
}
```

Un seul listener est actif à la fois : rappeler `attach` remplace le précédent. Appelle
`detach()` pour arrêter de recevoir les événements et te désenregistrer du centre média.

## Fonctionnement

`MediaControlService` est un singleton qui délègue à un pont natif propre à chaque OS.
`isAvailable()` renvoie `true` uniquement quand ce pont s'est chargé correctement sur la
plateforme courante ; quand il renvoie `false`, chaque appel est un no-op, ce qui te permet de
protéger l'initialisation avec une seule vérification.

`configure()` définit l'identité du lecteur. Sous Linux, il enregistre le nom de bus MPRIS
(par défaut `org.mpris.MediaPlayer2.<appId>`, tiré de `NucleusApp.appId`) ; sur macOS,
l'identité provient du bundle de l'application hôte ; sur Windows, il utilise l'AUMID de
l'application. Les deux paramètres ont une valeur par défaut, donc `configure()` sans argument
est le cas courant.

Une fois configuré, pousse les métadonnées et l'état de lecture au fil des changements du
lecteur. Les commandes entrantes sont analysées depuis le pont natif et livrées à ton listener
sur l'EDT Swing.

<Callout type="info" title="Événements par plateforme">
Les commandes que l'OS peut renvoyer diffèrent selon la plateforme :

- **Linux (MPRIS)** émet toutes les variantes de `MediaControlEvent`.
- **macOS** émet `Play`, `Pause`, `Toggle`, `Next`, `Previous`, `Stop` et `SetPosition`.
- **Windows (SMTC)** émet `Play`, `Pause`, `Next`, `Previous`, `Stop`, `SeekBy` et `SetPosition`.
</Callout>

## Référence de l'API

### Définir les métadonnées

```kotlin
MediaControlService.setMetadata(
    MediaMetadata(
        title = "Track",
        artist = "Artist",
        album = "Album",
        coverUrl = "file:///tmp/cover.png",
        duration = 3 * 60 * 1_000L,
    ),
)
```

Chaque champ de `MediaMetadata` est nullable et vaut `null` par défaut. `duration` est en
millisecondes.

### Définir l'état de lecture

```kotlin
MediaControlService.setPlaybackState(
    MediaPlaybackState(
        status = MediaPlaybackStatus.PLAYING,
        positionMs = currentPositionMs,
    ),
)
```

`status` vaut `MediaPlaybackStatus.STOPPED`, `PAUSED` ou `PLAYING`. `positionMs` est optionnel
et vaut `null` par défaut. Pousse une mise à jour à chaque lecture, pause, arrêt et seek pour
que la barre de progression du système reste juste.

### Définir le volume

```kotlin
MediaControlService.setVolume(0.5)
```

`volume` est ramené dans l'intervalle `0.0`–`1.0`.

### Attacher et détacher

```kotlin
MediaControlService.attach { event -> /* gère */ }
MediaControlService.detach()
```

### Notes par plateforme

<Tabs items={["macOS", "Windows", "Linux"]}>
<Tab value="macOS">

Définis `LSApplicationCategoryType` à `public.app-category.music` dans l'`Info.plist` pour que
le widget Now Playing traite l'application comme un lecteur média. Les touches médias
matérielles livrent Play, Pause, Next et Previous quand l'application est le client
now-playing actif.

</Tab>
<Tab value="Windows">

L'intégration SMTC est automatique dès que tu appelles `configure()`. Le système considère le
dernier publisher comme source now-playing courante, et l'identité se résout via l'AUMID de
l'application.

</Tab>
<Tab value="Linux">

`configure()` enregistre le nom de bus MPRIS `org.mpris.MediaPlayer2.<appId>`, dérivé de
`NucleusApp.appId`. `playerctl` et les indicateurs de bureau découvrent le lecteur en
énumérant ce préfixe. Passe un `dbusName` personnalisé pour le remplacer.

</Tab>
</Tabs>

## Notes

- `setVolume(...)` est un no-op sur macOS — Now Playing n'a pas de canal de volume par
  application.
- Sous Linux sans D-Bus de session, `isAvailable()` renvoie `false` et chaque appel est un no-op.
