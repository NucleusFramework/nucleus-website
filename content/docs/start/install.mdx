Two paths to a running Nucleus app: the **demo installer** (60-second smoke test of the framework's capabilities) or the **Gradle plugin** (start your own project). Both work on macOS, Linux and Windows.

## TL;DR
- One-liner installers fetch a signed, notarized build of the Nucleus demo.
- The Gradle plugin lives on the [Gradle Plugin Portal](https://plugins.gradle.org/plugin/dev.nucleusframework) — no extra repository config.
- Prerequisites: JDK 17+, Gradle 8+, Kotlin 2.0+. JDK 25+ unlocks the AOT cache.

## Try the demo

<Tabs items={["macOS", "Linux", "Windows"]}>
<Tab value="macOS">

```bash
curl -fsSL https://nucleusframework.dev/install.sh | bash
```

Detects Apple Silicon vs Intel, downloads the matching DMG, installs to `/Applications`, and launches.

</Tab>
<Tab value="Linux">

```bash
curl -fsSL https://nucleusframework.dev/install-linux.sh | bash
```

Detects your architecture and package manager, then installs the matching `.deb` or `.rpm`.

</Tab>
<Tab value="Windows">

Download the installer from [GitHub Releases](https://github.com/NucleusFramework/Nucleus/releases) and run it.

</Tab>
</Tabs>

The demo exercises decorated windows, dark-mode detection, auto-update, taskbar progress and system integration — everything you can read about in [/docs/os](/docs/os) and [/docs/window](/docs/window). Its source lives in the [`example/`](https://github.com/NucleusFramework/Nucleus/tree/main/example) directory.

## Build it yourself

Clone the repo and run the same packaging task you'd use in your own project:

```bash
git clone https://github.com/NucleusFramework/Nucleus.git
cd Nucleus
./gradlew :example:packageDistributionForCurrentOS
```

Artifacts land in `example/build/compose/binaries/main/`.

## Prerequisites

| Requirement | Version | Notes |
|---|---|---|
| JDK | 17+ | JBR 17+ recommended for the AWT backend; JDK 25+ enables AOT cache. |
| Gradle | 8.0+ | Wrapper bundled with new projects. |
| Kotlin | 2.0+ | Compose Multiplatform requires Kotlin 2.x. |
| OS | macOS 12+, Windows 10+, Linux (glibc 2.31+) | Tao backend requires Wayland or X11 on Linux. |

<Callout type="tip" title="Pick the JDK">
For development, any JDK 17+ works. To ship a GraalVM binary, install a GraalVM 25 distribution and let the plugin invoke `native-image`. The plugin can also use the JetBrains Runtime (JBR) for the AWT backend.
</Callout>
