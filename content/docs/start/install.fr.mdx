Deux chemins vers une app Nucleus qui tourne : l'**installateur de démo** (test express des capacités du framework en 60 secondes) ou le **plugin Gradle** (pour démarrer ton propre projet). Les deux marchent sur macOS, Linux et Windows.

## L'essentiel
- Les installateurs one-liner récupèrent un build signé et notarisé de la démo Nucleus.
- Le plugin Gradle vit sur le [Gradle Plugin Portal](https://plugins.gradle.org/plugin/dev.nucleusframework) — aucune config de dépôt supplémentaire.
- Prérequis : JDK 17+, Gradle 8+, Kotlin 2.0+. JDK 25+ débloque le cache AOT.

## Essayer la démo

<Tabs items={["macOS", "Linux", "Windows"]}>
<Tab value="macOS">

```bash
curl -fsSL https://nucleusframework.dev/install.sh | bash
```

Détecte Apple Silicon ou Intel, télécharge le DMG correspondant, installe dans `/Applications` et lance.

</Tab>
<Tab value="Linux">

```bash
curl -fsSL https://nucleusframework.dev/install-linux.sh | bash
```

Détecte ton architecture et ton gestionnaire de paquets, puis installe le `.deb` ou le `.rpm` adéquat.

</Tab>
<Tab value="Windows">

Télécharge l'installateur depuis [GitHub Releases](https://github.com/NucleusFramework/Nucleus/releases) et lance-le.

</Tab>
</Tabs>

La démo couvre les decorated windows, la détection du dark mode, l'auto-update, la progression dans la barre des tâches et l'intégration système — tout ce que tu peux lire dans [/docs/os](/docs/os) et [/docs/window](/docs/window). Le code source est dans le dossier [`example/`](https://github.com/NucleusFramework/Nucleus/tree/main/example).

## La builder toi-même

Clone le dépôt et lance la même tâche de packaging que dans ton projet :

```bash
git clone https://github.com/NucleusFramework/Nucleus.git
cd Nucleus
./gradlew :example:packageDistributionForCurrentOS
```

Les artefacts apparaissent dans `example/build/compose/binaries/main/`.

## Prérequis

| Outil | Version | Notes |
|---|---|---|
| JDK | 17+ | JBR 17+ recommandé pour le backend AWT ; JDK 25+ active le cache AOT. |
| Gradle | 8.0+ | Wrapper fourni dans les nouveaux projets. |
| Kotlin | 2.0+ | Compose Multiplatform exige Kotlin 2.x. |
| OS | macOS 12+, Windows 10+, Linux (glibc 2.31+) | Le backend Tao demande Wayland ou X11 sur Linux. |

<Callout type="tip" title="Choisir le JDK">
Pour le dev, n'importe quel JDK 17+ convient. Pour livrer un binaire GraalVM, installe une distribution GraalVM 25 et laisse le plugin invoquer `native-image`. Le plugin peut aussi utiliser la JetBrains Runtime (JBR) pour le backend AWT.
</Callout>
