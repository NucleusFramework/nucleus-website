Nucleus se livre comme un plugin Gradle. On l'ajoute à le projet Kotlin et on peut cibler macOS, Linux et Windows depuis le même build.

## L'essentiel
- Le plugin vit sur le [Gradle Plugin Portal](https://plugins.gradle.org/plugin/dev.nucleusframework) — aucune config de dépôt supplémentaire.
- Prérequis : JDK 17+, Gradle 8+, Kotlin 2.0+. JDK 25+ débloque le cache AOT.

## Prérequis

| Outil | Version | Notes |
|---|---|---|
| JDK | 17+ | JBR 17+ recommandé pour le backend AWT ; JDK 25+ active le cache AOT. |
| Gradle | 8.0+ | Wrapper fourni dans les nouveaux projets. |
| Kotlin | 2.0+ | Compose Multiplatform exige Kotlin 2.x. |
| OS | macOS 12+, Windows 10+, Linux (glibc 2.31+) | Le backend Tao demande Wayland ou X11 sur Linux. |

<Callout type="tip" title="Choisir le JDK">
Pour le dev, n'importe quel JDK 17+ convient. Pour livrer un binaire GraalVM, installe une distribution GraalVM 25 et laisse le plugin invoquer `native-image`. Le plugin peut aussi utiliser la JetBrains Runtime (JBR) pour le backend AWT.
</Callout>
