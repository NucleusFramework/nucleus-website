Nucleus ships first-party Compose implementations of every major desktop design language. The same `DecoratedWindow` Composable, restyled by the toolkit module of your choice тАФ so a Mac user gets macOS, a Windows user gets Fluent, an Ubuntu user gets Yaru, and an IDE-like tool reaches for Jewel. No more Material paint job on Win32.

## TL;DR
- Same `DecoratedWindow` everywhere тАФ toolkits only change the visual style.
- Pick one per OS, or mix freely: every toolkit works on every platform.
- Swap toolkits by changing the Gradle dependency, not your UI code.
- Material 2 and Material 3 fallbacks are always available.

## How it works

Every toolkit module wraps the same `DecoratedWindow` from `nucleus-application` and substitutes its own `TitleBarStyle` and `DecoratedWindowStyle`. The Composables in your screens don't change тАФ only the wrapper at the top of your tree does. That's why mixing toolkits is cheap: ship a Fluent build on macOS for screenshots, ship Liquid Glass for users, with the same code.

The styles live in `decorated-window-core` as plain data (`TitleBarColors`, `TitleBarMetrics`, `DecoratedWindowMetrics`). Each toolkit module exposes a `remember*Style()` Composable that derives them from its source-of-truth theme (Jewel's `JewelTheme`, Material's `MaterialTheme`, etc).

<Callout type="info">
The toolkits style the window chrome. The widgets inside (buttons, lists, text fields) are whatever you choose тАФ Material 3, Jewel, or your own design system. Mix and match.
</Callout>
