Nucleus est un framework Kotlin pour créer des applications de bureau natives multiplateformes. Il s'appuie sur Compose Multiplatform et ajoute les décorations de fenêtre natives, l'intégration au système d'exploitation, la signature de code et les installateurs natifs. Tu configures tout le build via un seul DSL Gradle, et le runtime s'exécute soit comme une image native GraalVM, soit sur la JVM.

## Fonctionnalités

- **Un seul langage pour toutes les plateformes.** Partage ton code Kotlin avec tes apps Android, iOS et web, avec Compose UI par-dessus.
- **Backend Tao (nouveau en 2.0).** Fenêtrage natif en Rust avec prise en charge de Wayland, du multi-touch et du stylet.
- **Deux runtimes, un seul DSL.** Exécution en image native GraalVM, ou sur la JVM avec un cache AOT (JDK 25+).
- **Plus de 30 modules d'intégration OS.** Notifications, system tray, menus dock, dark mode, hotkeys globaux et barre des tâches, chacun ajouté comme dépendance optionnelle.
- **18 formats d'installateur.** DMG, PKG, MSI, AppX, DEB, RPM, AppImage, Snap et Flatpak entre autres, avec signature de code et notarisation.

## Quoi de neuf en 2.0

- **Backend Tao.** Remplace le backend AWT par des fenêtres natives pilotées en Rust. Voir [Backends](/docs/concepts/backends).
- **GraalVM Native Image.** Les métadonnées de reachability de chaque module Nucleus sont fournies et résolues automatiquement.
- **Quatre toolkits natifs.** macOS 26 (Liquid Glass), Fluent, Yaru et Jewel partagent le même `DecoratedWindow` avec un habillage différent.
- **18 formats d'installateur.** Désormais avec AppX/MSIX, Flatpak et Snap, avec publication sur les stores.
- **Cache AOT (JDK 25+).** Un chemin de démarrage JVM pré-chauffé.
