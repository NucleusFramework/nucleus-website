Le plugin Gradle Nucleus (`id("dev.nucleusframework")`) expose une seule extension top-level : `nucleus`. Elle reflète le DSL Compose Desktop de JetBrains et l'étend avec des formats, des surfaces de signature, des paramètres GraalVM, des pipelines store, et du wiring runtime. Cette page liste chaque propriété dans l'ordre où elle apparaît dans les sources du plugin.

## Top-level : `nucleus { }`

`dev.nucleusframework.NucleusExtension` :

- `val application: JvmApplication` — configuration app JVM (`nucleus { application { … } }`).
- `val nativeApplication: NativeApplication` — cibles macOS Kotlin/Native.
- `val dependencies: NucleusPlugin.Dependencies` — coordonnées / notations des modules runtime.

## `application { }` — `JvmApplication`

- `from(SourceSet)` / `from(KotlinTarget)`
- `disableDefaultConfiguration()`
- `dependsOn(vararg tasks: Task | String)`
- `fromFiles(vararg files: Any)`
- `var mainClass: String?`
- `val mainJar: RegularFileProperty`
- `var javaHome: String`
- `val args: MutableList<String>` / `args(vararg args: String)`
- `val jvmArgs: MutableList<String>` / `jvmArgs(vararg jvmArgs: String)`
- `val nativeDistributions: JvmApplicationDistributions` — voir ci-dessous
- `val buildTypes: JvmApplicationBuildTypes`
- `val graalvm: GraalvmSettings` — voir [GraalVM](#graalvm--graalvmsettings)

## `nativeDistributions { }` — `JvmApplicationDistributions`

| Propriété | Type | Notes |

| `modules: ArrayList<String>` | — | Défauts : `java.base, java.desktop, java.logging, java.net.http, jdk.accessibility, jdk.crypto.ec` |

| `var cleanupNativeLibs: Boolean` | — | Stripper les libs natives hors cible |

| `val trustedCertificates: ConfigurableFileCollection` | — | Certificats CA → `cacerts` du JDK livré (voir [certificats de confiance](/docs/packaging/trusted-certificates)) |
| `val protocols: MutableList<UrlProtocol>` | — | Protocoles de deep link |
| `targetFormats: ...` | hérité | Les 16 formats |

Méthodes :

- `fun fileAssociation(mimeType, extension, description, linuxIconFile?, windowsIconFile?, macOSIconFile?)`

### Enum `TargetFormat`
`Dmg`, `Pkg`, `Msi`, `Exe`, `Nsis`, `NsisWeb`, `AppX`, `Portable`, `Deb`, `Rpm`, `AppImage`, `Flatpak`, `Snap`, `Zip`, `Tar`, `SevenZip`, plus l'intermédiaire `RawAppImage`.

## `macOS { }` — `JvmMacOSPlatformSettings`

Étend `AbstractMacOSPlatformSettings`. Identification :

- `packageName: String?`, `packageVersion`, `packageBuildVersion`
- `dmgPackageVersion`, `dmgPackageBuildVersion`, `pkgPackageVersion`, `pkgPackageBuildVersion`
- `dockName: String?`, `setDockNameSameAsPackageName: Boolean = true`
- `bundleID: String?`, `appCategory: String?`, `minimumSystemVersion: String?`
- `installationPath: String? = "/Applications"`
- `iconFile`, `layeredIconDir` (répertoire `.icon` macOS 26+)

Signature & sandbox :

- `entitlementsFile: RegularFileProperty`, `runtimeEntitlementsFile`
- `provisioningProfile: RegularFileProperty`, `runtimeProvisioningProfile`
- `signing { … }` — voir [Signature](#macos--signing--macossigningsettings)
- `notarization { … }` — voir [Notarisation](#macos--notarization--macosnotarizationsettings)

macOS 26 :

- `macOsSdkVersion: String? = "26.0"` — `vtool` patche le `LC_BUILD_VERSION` du launcher Mach-O pour activer Liquid Glass au SDK 26.

Composition :

- `launchAgents { agent(plistName) { bundleProgram; arguments; startInterval } }`
- `infoPlist { extraKeysRawXml }`
- `dmg { … }` — voir [DMG](#macos--dmg--dmgsettings)

### `macOS.signing { }` — `MacOSSigningSettings`
- `sign: Property<Boolean>` (défaut `false`)
- `identity: Property<String>`
- `keychain: Property<String>`
- `prefix: Property<String>`

### `macOS.notarization { }` — `MacOSNotarizationSettings`
Trois modes mutuellement exclusifs :

- Apple ID : `appleID`, `password`, `teamID`
- Profil keychain `notarytool` : `keychainProfile`, optionnel `keychainPath`
- Clé API App Store Connect : `apiKey`, `apiKeyId`, `apiIssuer`

### `macOS.dmg { }` — `DmgSettings`
- `title: String?`, `iconSize: Int?`, `iconTextSize: Int?`
- `format: DmgFormat?` (`UDRW`, `UDRO`, `UDCO`, `UDZO`, `UDBZ`, `ULFO`)
- `size: String?`, `shrink: Boolean?`, `sign: Boolean`
- `background: RegularFileProperty`, `backgroundColor: String?`
- `icon: RegularFileProperty`, `badgeIcon: RegularFileProperty`
- `window { x; y; width; height }`
- `content(x, y, type: DmgContentType?, name: String?, path: String?)` — `DmgContentType` : `File`, `Link`, `Dir`

## Notes

Le plugin injecte une ressource `nucleus/nucleus-app.properties` dans l'app packagée contenant `app.id`, `app.version`, `app.vendor`, `app.aumid`, `startup.task.id`, etc. L'objet `NucleusApp` au runtime les lit au démarrage — c'est ainsi que la config DSL coule vers le code lib.
