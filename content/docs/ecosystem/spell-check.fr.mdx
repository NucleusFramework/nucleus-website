La correction orthographique est un domaine profond et lourd en locales. La bonne réponse est d'utiliser le même moteur que l'OS utilise partout ailleurs — et l'écosystème Kotlin a déjà le wrapper.

## TL;DR
- Utilise [**PlatformSpellCheckerKt**](https://github.com/Wavesonics/PlatformSpellCheckerKt) par [@Wavesonics](https://github.com/Wavesonics).
- Wrappe `NSSpellChecker` sur macOS, `ISpellChecker` sur Windows (depuis Windows 8), et `enchant`/`hunspell` sur Linux.
- Suggestions, détection de langue, dictionnaires custom.

## Installation

```kotlin
dependencies {
    implementation("com.darkrockstudios:platform-spellchecker:<version>")
}
```

## Démarrage rapide

```kotlin
val checker = PlatformSpellChecker()

val misspellings = checker.check("This is a sentance with errors")
// → [Misspelling(word = "sentance", suggestions = ["sentence", ...])]
```

## Comment ça marche

- **macOS** — `NSSpellChecker`, le même moteur que TextEdit et les champs de texte système.
- **Windows** — Windows Spell Check API (`ISpellChecker`), disponible depuis Windows 8.
- **Linux** — `enchant` / `hunspell`.

On obtient des suggestions, de la détection de langue, et le support de dictionnaires custom — même comportement que l'utilisateur voit déjà partout dans son OS.

## Pourquoi Nucleus ne livre pas ça

La correction orthographique est lourde en locales et bénéficie d'une bibliothèque focalisée et bien testée. PlatformSpellCheckerKt wrappe déjà les bons moteurs natifs sans limitation JVM que Nucleus aurait à contourner.
